# Analytics and Forecasting

## Business purpose
This module adds a lightweight planning layer to the ERP-style analysis by creating baseline revenue forecasts and a separately labeled cash collection forecast.

## Input files used
- `03_FI_Module/outputs/revenue_trend.csv`
- `03_FI_Module/outputs/cash_collections.csv`
- Source data regenerated by `scripts/analytics_pipeline.py`

## Analysis performed
- Naive, three-month moving average, and linear trend baselines for monthly revenue.
- Three-month holdout comparison using MAPE and MAE for revenue baselines.
- Three-period cash collection forecast using a three-month moving average of historical collections.

## Output files generated
- `07_Analytics_Forecasting/outputs/forecast_report.md`
- `07_Analytics_Forecasting/outputs/revenue_forecast.csv`
- `07_Analytics_Forecasting/outputs/cash_collection_forecast.csv`
- `07_Analytics_Forecasting/outputs/forecast_metrics.csv`
- `07_Analytics_Forecasting/outputs/revenue_forecast.svg`

## ERP/SAP relevance
The module is inspired by ERP embedded analytics and planning workflows where finance teams review historical actuals before preparing short-term planning baselines.

## Management decisions supported
- Use baseline trend expectations for planning conversations.
- Compare actual revenue and cash collection movement.
- Separate revenue forecasting from cash collection timing so working-capital discussions are clearer.

## Forecasting caution
The forecast is intentionally simple. The dataset is synthetic and short, so MAPE and MAE are directional learning metrics only, not proof of production-grade forecast accuracy.
